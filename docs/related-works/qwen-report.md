# From Black Box to BPF: Formalizing the Path from Transformer Detection to Kernel-Level Enforcement

## System Call Sequence Analysis as the New Host-Level Security Boundary

The rapid adoption of microservice-based architectures and containerization technologies like Docker has fundamentally altered the threat landscape of modern computing environments [[167](https://arxiv.org/html/2510.12399v1), [168](https://link.springer.com/article/10.1007/s10462-025-11458-6)]. As these containerized services share the host operating system's kernel, the traditional security perimeter defined by firewalls and network segmentation becomes porous [[10](https://www.sciencedirect.com/org/science/article/pii/S1546221826000615)]. Malicious activities originating within one container can now directly impact the host kernel and other co-resident containers, rendering network-centric defense mechanisms insufficient [[10](https://www.sciencedirect.com/org/science/article/pii/S1546221826000615), [16](https://pmc.ncbi.nlm.nih.gov/articles/PMC9962260/)]. This paradigm shift necessitates a move from analyzing external network traffic to scrutinizing the internal behavioral patterns of processes executing on the host. The primary interface for this scrutiny is the system call (syscall) trace, which represents the communication channel between user-space applications and the Linux kernel [[23](https://pmc.ncbi.nlm.nih.gov/articles/PMC12608121/), [54](https://www.linkedin.com/pulse/root-cause-analysis-ebpf-practical-guide-sres-linux-engineers-akande-u7hhc)]. Consequently, the syscall interface has become the new, critical security boundary for enforcement [[16](https://pmc.ncbi.nlm.nih.gov/articles/PMC9962260/)].

System calls serve as the fundamental building blocks of an application's runtime behavior. Actions such as opening a file, making a network connection, or creating a new process are all executed through specific syscalls [[123](https://www.sciencedirect.com/science/article/pii/S1389128625009028?dgcid=rss_sd_all)]. By capturing and analyzing the sequence of these events, it becomes possible to model the "normal" operational profile of an application [[6](https://dl.acm.org/doi/full/10.1145/3736755)]. Deviations from this established baseline can indicate malicious intent. For example, an attack chain designed to escalate privileges might involve a series of seemingly innocuous syscalls, such as resolving file paths, hijacking symbolic links, and accessing sensitive files in unexpected locations [[69](https://www.mdpi.com/2227-7390/14/6/991)]. Similarly, an attacker compromising a container might use container management APIs, such as those for Docker, to stage further attacks, a pattern that can be detected by monitoring relevant API endpoint interactions [[169](https://www.researchgate.net/publication/397365691_Temporal_Analysis_Framework_for_Intrusion_Detection_Systems_A_Novel_Taxonomy_for_Time-Aware_Cybersecurity)]. This approach moves security analysis from a reactive, signature-based model to a proactive, behavior-based one, capable of detecting novel and zero-day threats by identifying anomalous execution patterns rather than matching known signatures [[20](https://www.sciencedirect.com/science/article/pii/S2542660525002422), [51](https://arxiv.org/pdf/2507.04752)].

A significant technical challenge in applying machine learning to this domain is the representation of raw syscall data into a format suitable for deep learning models. System call traces are inherently sequential and high-dimensional. To make this data tractable, researchers employ advanced preprocessing techniques. One such method is Byte-Pair Encoding (BPE), a subword tokenization algorithm originally developed for natural language processing [[164](https://dl.acm.org/doi/full/10.1145/3660769), [165](https://openreview.net/pdf/d34346f800285b0335d8f3781c83137c0f430321.pdf)]. BPE works by iteratively merging the most frequent pairs of bytes in a dataset, effectively building a vocabulary of common byte sequences [[165](https://openreview.net/pdf/d34346f800285b0335d8f3781c83137c0f430321.pdf)]. This process allows for the efficient compression of long syscall sequences and their conversion into a stream of discrete tokens that can be fed into architectures like Transformers [[108](https://arxiv.org/html/2511.22095v1)]. Datasets specifically designed for this purpose, such as MH-1M, provide pre-computed BPE tokenizations alongside structural metadata, underscoring the critical role of this step in enabling sequence modeling [[2](https://pmc.ncbi.nlm.nih.gov/articles/PMC12873278/), [110](https://www.researchgate.net/publication/398134523_Binary-30K_A_Heterogeneous_Dataset_for_Deep_Learning_in_Binary_Analysis_and_Malware_Detection)]. By transforming raw binary data or byte-level sequences into a standardized tokenized format, infrastructure is created that is purpose-built for transformer architectures, facilitating efficient attention mechanisms and robust sequence analysis [[108](https://arxiv.org/html/2511.22095v1)]. The ability to accurately capture and represent these temporal dependencies is the cornerstone upon which effective anomaly detection systems are built, as the true nature of a multi-stage attack often lies not in a single syscall but in the complex relationships between multiple syscalls over time [[67](https://www.researchgate.net/publication/369516073_Euler_Detecting_Network_Lateral_Movement_via_Scalable_Temporal_Link_Prediction), [131](https://www.semanticscholar.org/paper/Transformer-based-knowledge-distillation-for-system-AL-Nomasy-Alamri/83f93118c3bbadbdc8b7de4182ae27c56dfc2d04)].

## Transformer Architectures for Detecting Complex, Multi-Stage Attacks

In response to the need for sophisticated behavioral analysis, deep learning models have become central to modern intrusion detection systems (IDS) [[7](https://arxiv.org/html/2504.07839v3), [84](https://www.researchgate.net/publication/389834193_Transformer-based_knowledge_distillation_for_explainable_intrusion_detection_system)]. Among various architectures, the Transformer has emerged as a particularly powerful tool for detecting complex, multi-stage attacks by virtue of its ability to model long-range dependencies within system call sequences [[17](https://arxiv.org/html/2408.07583v2), [44](https://arxiv.org/html/2408.07583v1)]. Unlike Recurrent Neural Networks (RNNs), which process sequences sequentially and can struggle with vanishing gradients over long distances, or Convolutional Neural Networks (CNNs), which excel at local feature extraction but may miss global context, the Transformer's self-attention mechanism allows it to weigh the importance of all previous events in a sequence simultaneously [[29](https://media.neliti.com/media/publications/632790-real-time-intrusion-detection-leveraging-8c3e0047.pdf), [166](https://escholarship.org/content/qt92s294t8/qt92s294t8_noSplash_8d468e53681151634aa48a1ca8e601b0.pdf?t=s1abtu)]. This capability is crucial for identifying subtle correlations between disparate actions that, when combined, constitute a malicious activity. For instance, an attacker might perform reconnaissance using `getuid` and `uname` syscalls early in the attack chain, followed much later by a privilege escalation attempt using `setuid`. A standard IDS might miss this link, but a well-trained Transformer model can learn the latent relationship between these temporally distant events, flagging the entire sequence as anomalous [[166](https://escholarship.org/content/qt92s294t8/qt92s294t8_noSplash_8d468e53681151634aa48a1ca8e601b0.pdf?t=s1abtu)].

Empirical studies have validated the effectiveness of Transformer-based models in this domain. Researchers have proposed novel architectures specifically tailored for cyber-threat detection. For example, the IDS-MTran model utilizes a multi-scale Transformer to capture diverse traffic features, broadening its analytical capabilities [[24](https://www.nature.com/articles/s41598-024-74214-w)]. Another approach, KD-XVAE, employs a Variational Autoencoder (VAE) in conjunction with a Transformer to model system call data for in-vehicle networks, demonstrating the architecture's versatility beyond traditional network traffic [[133](https://www.researchgate.net/publication/384886661_Transforming_In-Vehicle_Network_Intrusion_Detection_VAE-based_Knowledge_Distillation_Meets_Explainable_AI)]. Vision Transformers (ViTs), while primarily designed for image data, are also being explored for cyber-threat detection tasks, suggesting a trend towards adapting powerful vision-centric architectures for security analytics [[25](https://www.sciencedirect.com/science/article/pii/S0167404824002293), [167](https://arxiv.org/html/2510.12399v1)]. Furthermore, hybrid models that combine Transformers with other architectures, such as CNNs and Bi-directional LSTMs (BLSTMs), have been shown to better handle complex features and long-sequence dependencies, achieving improved performance in intrusion detection tasks [[132](https://pmc.ncbi.nlm.nih.gov/articles/PMC12074129/)]. These advancements highlight a clear research trajectory towards leveraging the full power of attention-based models to understand the intricate dynamics of modern cyberattacks.

Despite their superior analytical capabilities, the deployment of large-scale Transformer models in production environments presents significant hurdles. Their high computational demands lead to substantial latency during inference, which is often unacceptable for real-time security applications that require immediate threat response [[45](https://arxiv.org/html/2507.04752v1), [84](https://www.researchgate.net/publication/389834193_Transformer-based_knowledge_distillation_for_explainable_intrusion_detection_system)]. Additionally, these models are frequently treated as "black boxes," meaning their decision-making processes are opaque and difficult for human analysts to interpret [[43](https://pmc.ncbi.nlm.nih.gov/articles/PMC12923649/)]. This lack of explainability erodes trust and complicates incident response, as security teams cannot easily verify why a particular event was flagged as malicious or determine the precise cause of a false positive [[84](https://www.researchgate.net/publication/389834193_Transformer-based_knowledge_distillation_for_explainable_intrusion_detection_system)]. These two challenges—computational overhead and opacity—are the primary motivations for exploring techniques that can translate the knowledge captured by a powerful Transformer "Teacher" into a more practical and transparent form suitable for deployment.

## Knowledge Distillation as a Bridge to Efficient and Interpretable Models

To bridge the gap between the high-performance but impractical Transformer models and the requirements of real-world deployment, Knowledge Distillation (KD) has emerged as a pivotal technology [[92](https://arxiv.org/pdf/2511.03799), [173](https://arxiv.org/pdf/2206.13130)]. Introduced by Hinton et al., KD is a model compression technique where a smaller, faster "Student" model is trained to mimic the behavior of a larger, pre-trained "Teacher" model [[98](https://arxiv.org/pdf/2504.04378), [118](https://arxiv.org/html/2504.14772v1)]. The core idea is to transfer the nuanced knowledge learned by the Teacher—not just its final predictions but also the soft probability distributions over classes—to the Student [[118](https://arxiv.org/html/2504.14772v1), [179](https://aclanthology.org/people/lili-mou/unverified/)]. This process effectively "distills" the expertise of the complex model into a more compact and efficient structure. In the context of cybersecurity, the Teacher is typically a large Transformer model trained extensively on system call sequences to achieve high accuracy in anomaly detection [[27](https://pmc.ncbi.nlm.nih.gov/articles/PMC12215491/), [28](https://www.mdpi.com/2227-7390/12/24/4009)]. The Student, in contrast, is designed to be lightweight, computationally inexpensive, and ideally, interpretable.

The primary benefit of KD is addressing the computational overhead of deep learning models. High-capacity models incur significant latency, which is untenable for real-time intrusion detection [[84](https://www.researchgate.net/publication/389834193_Transformer-based_knowledge_distillation_for_explainable_intrusion_detection_system)]. By distilling the knowledge into a smaller Student model, the resulting system can achieve low-latency inference, making it viable for production environments where timely responses are critical [[28](https://www.mdpi.com/2227-7390/12/24/4009), [141](https://www.nature.com/articles/s41598-025-21261-6)]. Frameworks like DistillGuard and DistillMal exemplify this approach, aiming to create efficient IDS solutions without a substantial loss in detection accuracy [[27](https://pmc.ncbi.nlm.nih.gov/articles/PMC12215491/), [28](https://www.mdpi.com/2227-7390/12/24/4009)]. This enables the deployment of sophisticated detection logic on resource-constrained edge devices or within high-throughput cloud environments [[43](https://pmc.ncbi.nlm.nih.gov/articles/PMC12923649/), [88](https://dl.acm.org/doi/10.1016/j.neucom.2025.130746)]. The efficiency gains are not merely academic; they are essential for maintaining system performance while continuously monitoring for threats.

Beyond computational efficiency, KD provides a pathway to interpretability. Instead of attempting to reverse-engineer the opaque weight matrices of a Transformer, researchers can use KD to transfer knowledge into a white-box model whose internal logic is naturally understandable to humans [[122](https://arxiv.org/pdf/2403.09053)]. For instance, the distilled knowledge can be used to train a decision tree, a set of logical rules, or a fuzzy inference system [[79](https://arxiv.org/html/2403.01954v1), [152](https://pmc.ncbi.nlm.nih.gov/articles/PMC7712003/), [154](https://www.sciencedirect.com/science/article/abs/pii/S0957417423023461)]. This creates a direct and verifiable link between the detection logic of the powerful Teacher model and the enforcement policies generated for the system. The process involves training the Student model to replicate the outputs of the Teacher, thereby inheriting its detection capabilities while presenting them in a transparent format. This dual benefit of efficiency and transparency makes KD a foundational component in the proposed pipeline, acting as the necessary intermediary that transforms a theoretical detection model into a practical, deployable security solution. Various distillation techniques have been explored, including feature-based KD, rationale distillation, and diversity-enhanced approaches, each aiming to optimize the knowledge transfer process for different objectives [[80](https://arxiv.org/pdf/2311.09612), [150](https://www.sciencedirect.com/science/article/abs/pii/S0957417425027757), [151](https://www.sciencedirect.com/science/article/abs/pii/S0306457325000019)].

| Technique | Description | Primary Application |
| :--- | :--- | :--- |
| **Standard Knowledge Distillation** | Trains a student model to mimic the output probabilities (soft labels) of a teacher model. [[92](https://arxiv.org/pdf/2511.03799), [98](https://arxiv.org/pdf/2504.04378), [118](https://arxiv.org/html/2504.14772v1)] | Model compression for low-latency inference. While standard KD focuses on output replication, more advanced techniques aim to preserve the internal structure and reasoning of the teacher model, potentially leading to even more faithful and efficient student models.

## Explainable AI for Generating Verifiable Security Policies

While Knowledge Distillation addresses the challenges of computational cost and opacity, it does not automatically produce the desired outcome of human-readable, verifiable security policies. This is the domain of Explainable Artificial Intelligence (XAI). XAI encompasses a range of techniques aimed at creating models that are transparent and whose decisions can be understood and trusted by human users [[162](https://www.kaggle.com/code/yatrikshah/hands-on-xai-with-shap-and-lime)]. In the context of this research goal, XAI plays a crucial role not just in providing post-hoc explanations for a model's predictions, but in actively generating the formal "if-then" rules that will constitute the final enforcement policy [[122](https://arxiv.org/pdf/2403.09053)]. The ultimate objective is to translate the abstract decision boundaries learned by a model into concrete, auditable rules that security analysts can inspect, validate, and confidently deploy in a production environment.

The process of generating these policies leverages the synergy between KD and XAI. After knowledge has been distilled into a white-box Student model—such as a decision tree, a shallow neural network, or a rule-based system—the inherent structure of this model can be analyzed to extract explicit rules [[152](https://pmc.ncbi.nlm.nih.gov/articles/PMC7712003/), [154](https://www.sciencedirect.com/science/article/abs/pii/S0957417423023461)]. For example, a decision tree naturally partitions the feature space based on a series of logical conditions, and these paths from root to leaf nodes can be directly translated into a set of if-then rules. This approach ensures that the generated policy is a faithful representation of the model's learned logic [[122](https://arxiv.org/pdf/2403.09053)]. Some research explores hybrid frameworks that combine deep learning models with fuzzy rules to enhance explainability from the outset [[154](https://www.sciencedirect.com/science/article/abs/pii/S0957417423023461)], while other work proposes specialized architectures like the Tree-Network-Tree (TNT) framework, which alternately transfers knowledge between a tree (for interpretation) and a network (for performance) [[152](https://pmc.ncbi.nlm.nih.gov/articles/PMC7712003/)]. This integrated methodology provides a systematic way to move from a complex, predictive model to a simple, prescriptive policy.

Several popular XAI methods are used to analyze and explain model behavior, though their utility in policy generation varies. LIME (Local Interpretable Model-agnostic Explanations) and SHAP (SHapley Additive exPlanations) are widely cited for providing local, post-hoc explanations by approximating a model's behavior around a specific prediction [[55](https://ieeexplore.ieee.org/document/10903225/), [56](https://journals.plos.org/plosone/article?id=10.1371/journal.pone.0286652), [163](https://arxiv.org/html/2305.02012v3)]. While invaluable for debugging and understanding individual instances, they are less suited for deriving global, overarching rules that govern system-wide behavior [[162](https://www.kaggle.com/code/yatrikshah/hands-on-xai-with-shap-and-lime)]. More targeted rule-extraction techniques are therefore preferred for this specific goal. The literature highlights a growing interest in eclectically extracting rules from deep neural networks to provide broader explainability for IDS systems [[32](https://www.researchgate.net/publication/389128343_A_systematic_review_on_the_integration_of_explainable_artificial_intelligence_in_intrusion_detection_systems_to_enhancing_transparency_and_interpretability_in_cybersecurity)]. To ensure the quality of these extracted rules, a systematic evaluation framework has been proposed, incorporating metrics such as descriptive accuracy (how well the rules describe the model), sparsity (simplicity of the rules), fidelity (faithfulness to the original model), coverage (how many instances the rules apply to), and precision (how accurate the rules are) [[30](https://ieeexplore.ieee.org/iel8/6287639/10820123/11222568.pdf)]. The application of these rigorous evaluation standards is essential to build confidence that the generated policies are both effective and reliable, forming a solid foundation for automated, kernel-level enforcement.

## Kernel-Level Enforcement with eBPF for Real-Time Defense

Once an interpretable and effective policy has been generated, the final and most critical stage is its enforcement at the point of greatest impact: the kernel. The extended Berkeley Packet Filter (eBPF) has emerged as the enabling technology that makes this real-time, low-latency enforcement feasible and secure [[10](https://www.sciencedirect.com/org/science/article/pii/S1546221826000615), [149](https://cymulate.com/blog/ebpf_hacking/)]. Originally designed for packet filtering, eBPF is now a powerful virtual machine that allows sandboxed programs to run inside the Linux kernel [[10](https://www.sciencedirect.com/org/science/article/pii/S1546221826000615), [54](https://www.linkedin.com/pulse/root-cause-analysis-ebpf-practical-guide-sres-linux-engineers-akande-u7hhc)]. These programs, written in a restricted C-like language and compiled to bytecode, can be attached to various kernel "hooks"—events such as system calls, network packet transmission, or function entry/exit points [[5](https://ieeexplore.ieee.org/iel8/6287639/6514899/11366217.pdf), [148](https://www.ateam-oracle.com/from-kernel-to-logging-analytics-by-way-of-ebpf-a-high-level-overview)]. This capability allows for unprecedented observability and control over the system's behavior with minimal performance overhead [[23](https://pmc.ncbi.nlm.nih.gov/articles/PMC12608121/), [147](https://pdfcoffee.com/learning-ebpf--pdf-free.html)].

The relevance of eBPF to this research goal is profound. It provides the exact mechanism needed to implement the security policies derived from the ML pipeline. An eBPF program can be attached to the `tracepoint/sched/sched_switch` to monitor process switches, or to the `sys_enter_*` tracepoints to intercept every system call before it reaches the kernel's core functionality [[54](https://www.linkedin.com/pulse/root-cause-analysis-ebpf-practical-guide-sres-linux-engineers-akande-u7hhc)]. When a syscall occurs, the eBPF program executes, inspecting the syscall number and its arguments. It can then consult a map stored in the kernel (a key-value store) containing the rules generated by the XAI process. If the observed behavior matches a rule that constitutes a violation, the eBPF program can take immediate action, such as dropping the syscall, logging the event, or triggering a response from a user-space agent [[5](https://ieeexplore.ieee.org/iel8/6287639/6514899/11366217.pdf), [12](https://ieeexplore.ieee.org/iel7/6287639/10005208/10138542.pdf)]. This entire process happens within the kernel, avoiding the costly context switches between user space and kernel space that plague traditional user-space agents, thereby ensuring the low-latency enforcement required for real-time security [[23](https://pmc.ncbi.nlm.nih.gov/articles/PMC12608121/)]. Tools like `opensnoop`, which uses eBPF to trace `open()` syscalls in real-time, demonstrate the practical utility and efficiency of this approach [[54](https://www.linkedin.com/pulse/root-cause-analysis-ebpf-practical-guide-sres-linux-engineers-akande-u7hhc)].

Frameworks and tools are already leveraging eBPF for security purposes, validating its suitability for this role. DeSFAM (Dynamic eBPF-driven Syscall Filtering and Anomaly Mitigation) uses eBPF to dynamically filter syscalls based on observed behavior, enforcing least-privilege principles [[128](https://ieeexplore.ieee.org/iel8/6287639/10820123/11095719.pdf)]. Optimus similarly employs eBPF to identify essential syscalls for containers and enforce dynamic filtering [[130](https://www.researchgate.net/publication/379225373_Optimus_association-based_dynamic_system_call_filtering_for_container_attack_surface_reduction)]. These examples show that eBPF can be used to construct a closed-loop system for monitoring, analysis, and response at the kernel level [[5](https://ieeexplore.ieee.org/iel8/6287639/6514899/11366217.pdf)]. However, deploying eBPF is not without its challenges. The eBPF Virtual Machine has strict limitations on instruction count and memory access to prevent instability, and every program must pass a verifier that ensures it is safe to run (e.g., it cannot loop infinitely or access invalid memory) [[89](https://arxiv.org/html/2410.00026v1), [148](https://www.ateam-oracle.com/from-kernel-to-logging-analytics-by-way-of-ebpf-a-high-level-overview)]. Careful design and optimization are required to implement complex logic without exceeding these constraints, and while performance impact is generally low, it is not zero and must be measured in any production deployment [[106](https://www.researchgate.net/publication/393211756_The_impact_of_using_eBPF_technology_on_the_performance_of_networking_solutions_in_a_Kubernetes_cluster), [107](https://dev.to/yunwei37/measuring-function-latency-with-ebpf-2ogk)]. Despite these limitations, eBPF stands as the indispensable execution layer that brings the theoretical pipeline to life, providing the necessary speed and integration to defend against threats in microseconds, not seconds.

## Empirical Challenges and Future Research Directions

While the conceptual pipeline integrating Transformer-based detection, Knowledge Distillation, XAI, and eBPF enforcement is theoretically compelling, the provided literature reveals significant gaps and challenges that hinder its practical realization. A primary obstacle is the scarcity of high-quality, publicly available datasets. Most research in this area relies on synthetic data generation or repurposed network intrusion datasets like NSL-KDD, which may not accurately reflect the complexities and nuances of system call sequences in real-world containerized environments [[4](http://dl.acm.org/doi/full/10.1145/3638552?trk=public_post_comment-text), [46](https://arxiv.org/html/2509.03442v1)]. The development of specialized datasets, such as MH-1M for Android malware, which includes pre-tokenized data, would be a major catalyst for progress, enabling more robust empirical validation and benchmarking [[2](https://pmc.ncbi.nlm.nih.gov/articles/PMC12873278/), [110](https://www.researchgate.net/publication/398134523_Binary-30K_A_Heterogeneous_Dataset_for_Deep_Learning_in_Binary_Analysis_and_Malware_Detection)]. Without realistic data, it is difficult to assess the true generalizability and effectiveness of the proposed models and pipelines.

Another critical challenge is adversarial robustness. The entire system, from the Teacher model to the final eBPF policy, is vulnerable to adversarial attacks. Model Extraction Attacks (MEAs) pose a risk where an adversary could query a deployed model to reconstruct its logic, undermining its novelty [[50](https://arxiv.org/html/2508.15031v2)]. Furthermore, Universal Adversarial Perturbations (UAPs) could be crafted to fool the IDS with minimal modifications to the input data, evading detection entirely [[48](https://arxiv.org/pdf/2511.18223)]. The resilience of both the Transformer-based Teacher and the distilled Student model against such sophisticated attacks remains a critical but under-explored area of research. Ensuring the integrity of the system requires developing defenses against these potential exploits.

Furthermore, there exists a "faithfulness gap" in the translation from model to policy. Even with XAI, the explanations or rules generated can sometimes be misleading or inaccurate, a phenomenon akin to the "hallucination" observed in Large Language Models (LLMs) [[33](https://arxiv.org/html/2408.04342v1)]. An XAI method might produce a plausible-sounding explanation that does not truly reflect the model's decision-making process. Ensuring the fidelity of the extracted rules is paramount for building trustworthy security systems. Rigorous evaluation frameworks, such as the one proposed with metrics like fidelity and stability, are necessary to validate the quality of the generated policies [[30](https://ieeexplore.ieee.org/iel8/6287639/10820123/11222568.pdf)]. Finally, the integration of these disparate components—Transformer training, KD, XAI rule extraction, and eBPF deployment—into a cohesive, end-to-end system represents a significant engineering challenge [[100](https://ieeexplore.ieee.org/iel7/6287639/10380310/10540566.pdf)]. The complexity of managing this full stack is a practical barrier to adoption. Addressing these issues—data scarcity, adversarial robustness, faithfulness of explanations, and system integration—is essential for moving this promising research direction from theory to practice.
